# Admin 디자인 시스템 페이지 구현 계획

## 목표

- Admin 카테고리에 **디자인 시스템** 전용 페이지(`/admin/design-system`) 추가
- Storybook 없이 **같은 앱 내**에서 `@/components/ui` 컴포넌트를 import해 **실제 렌더링**으로 UI 샘플 제공
- **스타일 가이드** 내용(색상, 타이포그래피, 간격) + **확장 내용**(사용 규칙 Do/Don't, 접근성, 컴포넌트별 설명)을 한 페이지에 구성

## 현재 구조 참고

- Admin 메뉴: [components/layout/Sidebar.tsx](../components/layout/Sidebar.tsx) 339–384줄. `CategoryType.ADMIN`일 때 대시보드/회원 관리/공지사항 관리 링크를 하드코딩으로 렌더.
- Admin 페이지: [app/(dashboard)/admin/dashboard/page.tsx](../app/(dashboard)/admin/dashboard/page.tsx)와 동일 레이아웃(제목, 설명, 카드/그리드).
- 색상 토큰: [app/globals.css](../app/globals.css) `:root` / `.dark`의 `--penta-indigo`, `--penta-blue`, `--primary`, `--destructive` 등.
- UI 컴포넌트: [components/ui/](../components/ui/) 에 button, input, card, badge, alert, select, label, textarea, checkbox 등 (Shadcn 기반). Button은 [components/ui/button.tsx](../components/ui/button.tsx) 참고로 variant/size 다수.

## 1. 라우트 및 Sidebar 메뉴 추가

- **경로:** `app/(dashboard)/admin/design-system/page.tsx` 신규 생성.
- **Sidebar:** [components/layout/Sidebar.tsx](../components/layout/Sidebar.tsx) ADMIN 블록 내에 "디자인 시스템" 링크 추가. 기존 패턴과 동일하게 `href="/admin/design-system"`, `pathname.startsWith('/admin/design-system')`로 활성 스타일. 아이콘은 Lucide `Palette` 사용.
- **권한:** 기존 admin과 동일하게 middleware 및 세션으로 ADMIN만 접근(추가 설정 불필요).

---

## 2. 디자인 시스템 페이지 구조 (단일 페이지, 스크롤 섹션)

페이지는 **한 파일**에서 섹션별로 구분. 상단 제목/설명 아래 다음 순서로 구성.

- 제목 및 설명
- 1. 스타일 가이드: 색상 팔레트, 타이포그래피, 간격 및 radius
- 2. 컴포넌트: Button, Input, Card, Badge, Alert (각각 샘플 + 설명 + Do/Don't + 접근성)

---

## 3. 섹션별 구현 내용

### 3.1 스타일 가이드

- **색상**: Primary, Secondary, Destructive, Muted, Accent 및 Penta(Indigo, Blue, Sky, Green, Yellow) 색상 블록 + 변수명 표시.
- **타이포그래피**: Pretendard 기준. 제목(text-3xl font-bold), 섹션 제목(text-2xl font-semibold), 본문(text-base), 캡션(text-sm text-muted-foreground) 예시.
- **간격 및 Radius**: Tailwind p-2, p-4, p-6 및 rounded-md, rounded-lg, rounded-full 시각 예시.

### 3.2 컴포넌트 (UI 샘플 + 사용 규칙)

각 컴포넌트마다:

| 블록 | 내용 |
|------|------|
| **실제 샘플** | variant/size 등 여러 상태로 렌더 |
| **설명** | 용도와 사용 위치 (1–2문장) |
| **Do / Don't** | Do 2–3개, Don't 2–3개 문장 |
| **접근성** | 키보드 포커스, aria, 라벨 등 1–2문장 |

**포함 컴포넌트:** Button, Input, Card, Badge, Alert.

### 3.3 공통 UI

- 각 섹션은 Card + 제목(h2)으로 구분.
- Do/Don't는 리스트 형태로 표시.

---

## 4. 파일 구성

- **app/(dashboard)/admin/design-system/page.tsx**: 제목·설명, 스타일 가이드(색상/타이포/간격), 컴포넌트(Button/Input/Card/Badge/Alert) 샘플 + 설명 + Do·Don't + 접근성. `'use client'` 사용.

## 5. Sidebar 수정 상세

- "공지사항 관리" Link 다음에 "디자인 시스템" Link 추가.
- `href="/admin/design-system"`, 텍스트 "디자인 시스템", 아이콘 `Palette`.
- `pathname.startsWith('/admin/design-system')` 일 때 `text-[var(--penta-indigo)] dark:text-penta-sky bg-accent` 적용.

---

## 6. 주의사항

- 이 페이지는 **문서/가이드 성격**이므로 DB·API 연동 없이 정적 내용만으로 구현.
- 컴포넌트는 기존 `@/components/ui`를 그대로 import해 사용하므로, 스타일/토큰 변경 시 이 페이지에도 자동 반영됨.
- 접근성·Do/Don't 문구는 팀 합의 후 추후 수정 가능하도록 짧은 문장으로 작성.
