# 사이트 공지 페이지 (전역 유지보수·안내 모드)

도메인 이전, 점검, 업데이트 안내 등 **사이트 전체를 공지 화면으로 전환**할 때 사용합니다. **`SITE_NOTICE_MODE` 환경 변수**로 켜고 끄며, 공지 모드가 켜지면 **`/notice` 외 대부분의 경로는 접근할 수 없고** 공지 페이지로 리다이렉트됩니다.

---

## 1. 동작 요약

| 항목 | 설명 |
|------|------|
| **트리거** | `SITE_NOTICE_MODE`가 `true`, `1`, `on` 중 하나(대소문자·앞뒤 공백 무시) |
| **공지 URL** | `/notice` — [`app/notice/page.tsx`](../app/notice/page.tsx) |
| **미들웨어** | [`middleware.ts`](../middleware.ts) 최상단에서 공지 모드 분기 |
| **문구·구조** | [`lib/site-notice.ts`](../lib/site-notice.ts)의 `DEFAULT_SITE_NOTICE` + 선택적 환경 변수 |
| **UI** | [`components/notice/SiteNoticeView.tsx`](../components/notice/SiteNoticeView.tsx) |

공지 모드가 **꺼져 있을 때**도 `/notice`는 열립니다. 배포 전 문구·화면 확인용으로 쓸 수 있습니다.

---

## 2. 환경 변수

### 2.1 모드 전환

| 변수 | 값 | 설명 |
|------|-----|------|
| `SITE_NOTICE_MODE` | `true` / `1` / `on` | 전역 공지 모드 ON. 그 외·미설정이면 일반 서비스 |


로컬: **`.env.local`**. Vercel: **Project → Settings → Environment Variables**.  
`NEXT_PUBLIC_` 접두사가 없어 클라이언트 번들에 자동 주입되지 않습니다.

### 2.3 `.env` 예시

```env
SITE_NOTICE_MODE=true
```

---

## 3. 공지 모드 ON일 때 허용 경로

[`middleware.ts`](../middleware.ts)에서 아래는 **`NextResponse.next()`** 로 통과합니다.

| 경로 | 이유 |
|------|------|
| `/notice`, `/notice/*` | 공지 페이지 |
| `/_next/*` | Next 런타임·청크 |
| `/robots.txt`, `/sitemap.xml` | 정적 라우트 |
| `/api/auth/*` | Auth.js JSON. 막으면 세션 요청이 공지 HTML을 받아 `SessionProvider`에서 `ClientFetchError` 발생 가능 |

**그 외**는 `/notice`로 **302 리다이렉트**됩니다.

`matcher`는 `_next/static`, `_next/image`, `favicon.ico`, 일부 이미지 확장자 등을 제외합니다. 자세한 패턴은 `middleware.ts`의 `config.matcher`를 참고하세요.

## 4. 코드 구조

### 4.1 `lib/site-notice.ts`

- **`isSiteNoticeMode()`**: `SITE_NOTICE_MODE` 판별 (미들웨어·서버에서 동일 로직).
- **`getSiteNoticeConfig()`**: 기본값 + env 병합 결과 반환.
- **`SiteNoticeConfig`**: `eyebrow`, `title`, `descriptionParagraphs`, `highlight`, `footerParagraphs`, `primaryCta`, `secondaryCta`.
- **`SiteNoticeHighlight`**
  - **`comparison`**: `beforeBadge` / `beforeText` → `afterBadge` / `afterText`, 선택 `afterHref`(있으면 변경 후 텍스트가 링크).
  - **`callout`**: 선택 `title`, `lines[]`.
- **`DEFAULT_SITE_NOTICE`**: 운영 기본 문구·도메인 비교·CTA 정의. **강조 블록·CTA 내용은 env로 바꾸지 않고** 이 상수(또는 추후 JSON 등)에서만 조정하는 구조입니다.

### 4.2 페이지

- [`app/notice/page.tsx`](../app/notice/page.tsx): `getSiteNoticeConfig()` → `SiteNoticeView`.
- [`app/notice/layout.tsx`](../app/notice/layout.tsx): 래퍼 최소화.
- **`generateMetadata`**: `title`은 `config.title`과 동기, `robots: { index: false, follow: false }`, OG 제목에 `BRAND_EN` 병기.

### 4.3 `SiteNoticeView.tsx`

- **헤더**: `bg-penta-blue`, 점 패턴 오버레이, `eyebrow` + `title`.
- **본문**: `descriptionParagraphs`를 문단별 `<p>`로 표시.
- **강조**: `comparison` 또는 `callout` 렌더.
- **푸터**: `whitespace-pre-line`으로 **`\n` 줄바꿈** 반영. `https?://...`는 링크로 분리 렌더.
- **CTA 버튼**: `primaryCta` / `secondaryCta`용 JSX가 **현재는 주석 처리**되어 있을 수 있습니다. 버튼을 쓰려면 해당 블록 주석을 해제하고 `config`의 CTA와 연결하면 됩니다.

---

## 5. 배포 체크리스트 (Vercel)

1. Production(필요 시 Preview)에 `SITE_NOTICE_MODE=true` 등록.
2. 공지 종료 후 `SITE_NOTICE_MODE` 삭제 또는 `false` 등으로 비활성화 후 재배포(팀 정책에 따름).

---

## 6. 트러블슈팅

| 현상 | 조치 |
|------|------|
| Auth.js `ClientFetchError`, `Unexpected token '<'` | `/api/auth`가 공지 HTML로 리다이렉트되는지 확인. 기본 미들웨어는 `/api/auth` 허용. |
| 공지 모드에서 스타일·JS 깨짐 | `/_next` 통과 여부·`matcher` 제외 경로 확인. |
| env 변경이 반영되지 않음 | Vercel은 재배포 필요할 수 있음. 로컬은 dev 서버 재시작. |
